# LOVTARO 인스타 콘텐츠 생성

## 실행 흐름

### 0. 날짜 확인
사용자에게 아래를 질문해라:
> 몇월 몇일(요일)부터 몇월 몇일(요일)까지 생성할까요?

사용자가 날짜를 알려주면 해당 기간의 콘텐츠를 생성한다.
폴더명은 `YYYY-MM-DD_day` 형식 (예: `2026-04-20_mon`).

### 1. 사전 확인
- `content-output/` 기존 폴더 확인 (중복 방지)
- `public/images/cards-png/` 메이저 아르카나 목록 확인
- `public/images/mcards/` 마이너 아르카나 목록 확인
- **메모리 `project_lovtaro_card_usage.md` 반드시 Read** — 누적 사용 카드 목록 확인 후 미사용 카드 풀에서 배정
- 최근 1~2주 콘텐츠에서 사용한 카드 추가 확인 (중복 방지)

### 2. 주간 전략 수립 후 즉시 진행
아래를 정리해서 보여주되 **승인 대기 없이 바로 파일 생성을 시작**해라:
- 날짜별 포맷 배치 (shortform/carousel)
- shortform 유형 배치 (참여형/소개형 메이저/소개형 마이너) — 고정 주간 배치표 준수
- 날짜별 카드 배정 (메모리 `project_lovtaro_card_usage.md` 기반 미사용 카드 우선)
- 날짜별 주제/훅

### 3. 파일 생성 순서
아래 순서로 **TodoWrite로 진행상황 추적하면서** 실행:

1. 날짜별 폴더 구조 생성
2. 날짜별 copy.txt 작성
3. 참여형이 있으면 reply_templates.txt 작성
4. twitter.txt 작성
5. `scripts/templates/`에서 해당 포맷 템플릿을 `scripts/generate-{포맷}-{날짜}.mjs`로 **복사**한 뒤 카드/카피만 교체
6. 이미지 생성 스크립트 실행
7. **생성된 이미지 시각 검증 (필수)** — 각 장면을 Read 도구로 열어 다음 확인:
   - 한국어 렌더링 자소 정상 여부 (특히 `듦` 같은 겹받침 — KO_STACK 미적용 시 깨짐)
   - 텍스트 오버플로우/잘림 여부
   - 프레임·카드·텍스트 간격이 레퍼런스와 일치하는지
   - 코스믹 배경·달·별·프레임이 의도대로 그려졌는지
   - 문제 발견 시 스크립트 수정 후 재생성. 문제없어야 8번으로 진행
8. **`project_lovtaro_card_usage.md` 메모리 업데이트** — 이번 주에 사용한 메이저/마이너 카드(커버+본문+참여 전부) 추가 기록

## 코스믹 스타일 공통 요소 (릴스/숏폼 공통)

모든 릴스/숏폼 씬은 아래 공통 헬퍼를 사용한다:

### `cosmicDefs()` - SVG defs에 포함
- `cosmicBg` radialGradient: #1a0f38 → #140b2c → #0c0820 → #06040f
- `neb1` (오렌지), `neb2` (퍼플), `neb3` (골드) 성운 radialGradient
- `moonGlow` 달 주변 빛무리
- `moonMask` 크레센트 모양 마스크 (두 원 겹침)
- `softGlow` 텍스트용 블러 필터

### `cosmicBody(withMoon, starSeed)` - 배경 요소
- `cosmicBg` rect + 4개 성운 ellipse + ~500개 랜덤 별(mulberry32 시드) + 왼쪽 상단 달

### `drawFrame(x, y, w, h, strong)` - 카드 프레임
- 바깥 골드 rect (stroke 2.5) + 안쪽 가는 rect (gap 10)
- 4코너 장식 path (L자 모양)
- 색상: rgba(201,168,76,~) 계열

### `genStars(count, seed, ...)` - 별 생성 (mulberry32 seed 기반)

### 카드 이미지 처리
- `loadCard(slug, w, h)`: public/images/cards-png/{slug}.png 로드 후 리사이즈
- `roundImg(buf, w, h, r)`: 둥근 모서리 마스킹
- 프레임 안쪽에 카드 이미지를 sharp composite로 합성

  - 좋음: "다시 연락 올지 궁금한 사람 기준으로 / 카드 하나만 골라보세요"

  - 만다라/올-시잉 아이/골드 장식은 전부 동일 (사이트 정본 패턴 유지)
  - scheme별로 달라지는 것: 카드 내부 배경 그라디언트(다크 채도톤) + 별빛 팔레트 + 테두리 컬러 글로우 + 번호 라벨 accent색
  - scheme별 accent색은 `getSchemeAccent(scheme)` 헬퍼로 조회 (번호 라벨 fill에 사용)
  - 스킴 추가는 `SCHEMES` 객체에 `{bgTop, bgBot, darkOverlay, border, borderGlow, moonFill, stars, accent, starSeed}` 세트로 등록. 기존 API(`pickRandomSchemes`, `getSchemeAccent`, `colorCardBackDefs`, `colorCardBackSvg`) 시그니처 변경 금지

### B. 소개형 shortform (3장) — 마이너 아르카나 전용 / 확정안 v2 (2026-05-01 업데이트)

**scene01**: **카드 앞면 풀스크린 클로즈업** + 다크 그라디언트 + 한 줄 훅(핵심 단어 골드) + 작은 달

- 위쪽 다크 그라디언트(opacity 0.96 → 0)로 훅 가독성 영역 확보, 아래쪽도 동일하게 페이드

- 훅: **두 줄 한 카피**, 첫째 줄 평이 + 둘째 줄에 핵심 단어 `<tspan fill="#F0CC78" font-weight="700">` 골드 강조

## copy.txt 구조

### 줄바꿈 보존 규칙 (전 플랫폼 공통, 필수)

인스타·틱톡·유튜브는 캡션 붙여넣기 시 **빈 줄을 자동으로 제거**하기 때문에 사용자가 복붙하면 단락이 한 덩어리로 뭉친다. 이를 막으려면:

- **각 문장은 독립된 줄에 작성** (한 단락처럼 붙이지 말 것)
- **단락 사이 빈 줄은 반드시 `⠀`(U+2800, Braille Pattern Blank) 한 글자만 들어간 줄로 작성** — 시각적으로는 빈 줄로 보이지만 IG/틱톡이 보존
- 일반 공백·탭·완전 공백 줄은 IG가 strip하므로 사용 금지
- 해시태그 묶음 앞에도 `⠀` 줄을 한 번 넣어 본문과 분리

**좋은 예 (인스타 릴스 캡션)**:
```
지금 떠오르는 사람이 있다면
잠깐 멈추고 생각해보세요
⠀
1, 2, 3 중 마음 가는 번호를 골라보세요
고른 카드 기준으로 답글 달아드릴게요
⠀
✨ 매일 연애 타로 → @lovtarot_ 팔로우
⠀
#연애타로 #러브타로 #타로카드 #속마음타로 #짝사랑타로
```

**나쁜 예** (한 줄로 붙여넣기 + 일반 빈 줄):
```
지금 떠오르는 사람이 있다면 잠깐 멈추고 생각해보세요. 1, 2, 3 중 마음 가는 번호를 골라보세요.

#연애타로 #러브타로 ...
```

이 규칙은 **shortform·carousel copy.txt의 모든 캡션 섹션**(`[인스타 릴스 캡션]`, `[인스타 캐러셀 캡션]`, `[인스타 첫 댓글]`, `[유튜브 설명]`, `[유튜브 고정 댓글]`, `[틱톡 캡션]`, `[틱톡 첫 댓글]`)에 적용된다. `[유튜브 쇼츠 제목]`·`[커버 문구]` 같은 단일 줄 항목은 예외.

### shortform/copy.txt
```
[콘텐츠 주제]
[커버 문구]
[인스타 릴스 캡션]
[인스타 첫 댓글]
[유튜브 쇼츠 제목]
[유튜브 설명]
[유튜브 고정 댓글]
[틱톡 캡션]
[틱톡 첫 댓글]
```
- **[커버 문구]만 유지** — 한 줄 훅으로 전체 콘텐츠를 압축. scene01~05 문구는 copy.txt에 기록하지 않음 (이미지 안에 이미 있음, 스크립트 내 문자열이 단일 소스)
- **[인스타 릴스 캡션]**: 본문 + 해시태그 5개 (한국어 기반, 다양화) + **팔로우 CTA 1줄** (해시태그 바로 앞 또는 맨 끝) + **저장 또는 댓글 행동 유도 1줄 필수**
  - 저장 유도 예: "지금 떠오르는 사람이 있다면 저장해두세요. / 나중에 이 흐름이 맞았는지 다시 확인해보세요."
  - 댓글 유도 예 (참여형 한정): "댓글에 1, 2, 3만 남겨주세요. / 고른 카드 기준으로 답글 달아드릴게요."
  - 위치: 본문 마지막 줄 → 팔로우 CTA → 해시태그 순. 팔로우 CTA와 합쳐 2줄 초과 금지
- **[유튜브 쇼츠 제목]**: 반드시 끝에 `#shorts` 붙이기. **검색 키워드를 앞쪽에** 배치 (감성 제목보다 검색어 + 궁금증 조합이 노출 유리)
  - 키워드 후보: `상대방 속마음 타로` `재회 가능성 타로` `연락운 타로` `짝사랑 타로` `이별후 타로` `연애운 타로`
  - 좋음: "상대방 속마음 타로｜이 관계 지금 어디쯤 와 있을까? #shorts"
  - 좋음: "재회 가능성 타로｜그 사람 다시 연락 올까요? #shorts"
  - 나쁨: "이 관계 지금 어디쯤 와 있을까? 직감으로 골라보세요" (검색 키워드 부재)
  - 구분자는 `｜`(전각) 사용
- **[유튜브 설명]**: 본문 + **구독 CTA 1줄** + 해시태그 5개 (**한국어 3 + 영어 2**). 영어는 `#tarot` `#tarotreading` `#lovereading` `#dailytarot` `#tarotcards` 등에서 2개 선택
- **[유튜브 고정 댓글]**: 참여 유도 멘트 + **구독 CTA 1줄**
- **[틱톡 캡션]**: 본문(릴스보다 짧고 훅 중심, 1~2줄) + 해시태그 **5개** (**한국어 3 + 영어 2**: `#fyp` `#foryou` `#tarot` `#tarottok` `#lovereading` 등에서 2개 선택, `#fyp`는 가급적 포함) + **팔로우 CTA 1줄**
  - **틱톡 본문 톤 가이드**: 인스타·유튜브보다 **짧고 직접적, 가벼운 반말 톤** 허용 (러브타로 브랜드 고급 톤이라도 틱톡 첫 문장이 얌전하면 묻힘)
  - 좋음: "지금 생각나는 사람 기준으로 골라. 결과 소름일 수 있음."
  - 좋음: "그 사람 연락 기다리는 중이면 하나만 골라봐."
  - 나쁨: "다음 주 연애 흐름, 직감으로 한 장만 골라보세요 ✨" (인스타 톤 그대로 복사)
  - 단, **자극적 불안 마케팅·과장 금지** 원칙은 유지 (반말 ≠ 공포 자극)
- **[틱톡 첫 댓글]**: 프로필 링크 유도 멘트 1줄 (틱톡은 캡션 내 링크 비활성이라 고정 댓글로 유도)
- **파일 맨 끝**: `[틱톡 첫 댓글]` 아래 빈 줄 3개 추가 (복사 시 여백 확보)

#### 구독·팔로우 CTA 규칙
각 플랫폼 캡션/설명 끝(해시태그 앞 또는 맨 끝)에 **1줄만** 삽입. 2줄 이상 금지 (광고 느낌).

- **인스타**: `팔로우` + 핸들 **`@lovtarot_`** (예: `✨ 매일 연애 타로 → @lovtarot_ 팔로우`)
- **틱톡**: `팔로우` + 핸들 **`@lovtaro_`** (인스타와 핸들 다름 - 틱톡은 `t` 없음)
- **유튜브**: `구독` 단어 사용 (예: `📌 매일 연애 타로 업로드 - 구독하면 놓치지 않아요`)
- **톤**: "구독 부탁드려요" 같은 공손형 금지 → **혜택 제시형** ("매일 올라와요", "놓치지 않아요")
- **위치**: 본문 → CTA → 해시태그 순 (해시태그 뒤 CTA 금지 - 잘 안 읽힘)
- **다양화**: 매번 동일 문구 반복 금지, 유사 패턴 2~3개 로테이션
  - 인스타 후보: `팔로우하면 매일 리딩 ✨`, `매일 연애 타로 → @lovtarot_ 팔로우`, `다음 리딩 놓치지 마세요 @lovtarot_`
  - 틱톡 후보: `팔로우하면 매일 리딩 ✨`, `매일 연애 타로 → @lovtaro_ 팔로우`, `다음 리딩 놓치지 마세요 @lovtaro_`
  - 유튜브 후보: `매일 연애 타로 업로드 - 구독 🔔`, `구독하면 매일 리딩 알림`, `다음 리딩 보고싶다면 구독 🔔`

### 훅 공식 (필수 — 모든 포맷의 첫 장/도입 훅에 적용)

8가지 공식 중 1개 이상에 부합해야 함. 단순 카드 설명형(예: "컵의 2 카드가 알려주는 메시지") 금지.

**기본 5공식**

1. **상황 + 불안형**: "연락은 오는데, 마음은 안 보일 때"
2. **상대 행동 + 내 해석형**: "답장은 빠른데, 왜 거리는 느껴질까?"
3. **금지/경고형**: "이 신호를 놓치면 관계가 멀어질 수 있어요" (단, 공포 자극·과장 금지 — 관찰·체크리스트 톤 유지)
4. **선택 유도형**: "지금 떠오르는 사람 기준으로 하나만 골라보세요"
5. **시점 고정형**: "이번 주 안에 이 관계가 움직일까요?"

**데이터 기반 신규 3공식 (2026-05-05 추가)**

6. **위치 메타포형**: "그 사람 마음, 지금 어디쯤 와 있을까?" / "이 관계, 어느 단계에 머물러 있을까?" / "두 사람 마음의 거리, 얼마나 가까울까?"
   - 검증: 7,604회 (1·2·3 참여형 최고치)
   - 핵심: "어디쯤·어느 단계·거리" 같은 공간 메타포 + "지금" 시점 한정. 사용자가 "내 위치는 어디?" self-check 모드 진입
7. **양자택일 갈등형**: "호감일까 무관심일까" / "진심일까 빈말일까" / "기다림일까 포기일까"
   - 검증: 1,017회 (Devil 캐러셀 커버), 캐러셀·릴스 모두 강함
   - 핵심: 두 가지 핵심 갈등을 단어 2개로 압축. 추상어 금지 (예: "사랑일까 우정일까" X → "진심일까 빈말일까" O)
8. **시점+변화 예측형**: "다음 주, 둘 사이 흐름이 달라질까?" / "이번 주 안에 이 관계가 움직일까?" / "곧 다가올 변화는 무엇일까?"
   - 검증: 3,368회 (Sunday Preview)
   - 핵심: 미래 시점 + 변화 동사("달라질·움직일·다가올") 조합

**주제 선정 기준**: "그 사람 마음이 궁금해요" 같은 막연한 질문보다 **"상대가 X 행동을 하는데 이게 무슨 뜻이죠?"** 처럼 구체 행동·상황을 출발점으로 잡을 것 (공식 2·6 우선).

## 이미지 생성 스크립트 규칙

  - `getSchemeAccent(scheme)` — 번호 라벨 fill 색상 조회
  - `colorCardBackDefs()` / `colorCardBackSvg(cx, cy, scale, scheme)` — defs·카드 SVG 생성
  - 참여형 scene01에서만 사용. 소개형/캐러셀에는 정본(`card-back-svg.mjs`) 사용

### 레퍼런스 스크립트 (단일 소스 — `scripts/templates/`)

- 수치 변경이 필요하면 **템플릿 자체를 업데이트** (스킬 문서 수치 수정만 하면 드리프트 발생)
