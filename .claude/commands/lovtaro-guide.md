# 러브타로 가이드 작성 스킬 (프리미엄)

`lovtaro` 프로젝트의 `/guide` 섹션에 **퀄리티가 브랜드의 전부를 결정하는** 가이드 글 1편을 작성한다. 얇거나 AI스러운 글은 절대 발행하지 않는다. 한 편이 안 나오더라도, **기준 미달이면 기준까지 다시 쓴다**.

## 존재 이유

러브타로는 타로 서비스다. 독자는 "맞든 틀리든 해석이 마음에 닿는지"로 신뢰를 판단한다. 가이드 글은 그 신뢰의 첫 관문이다. 글 한 편이 얄팍하면:
- 검색 유입 독자가 3초 안에 이탈
- 브랜드 톤이 "그냥 그런 타로 블로그" 하나로 전락
- AdSense 심사 + E-E-A-T 평가에서 감점
- 리딩 서비스 신뢰도 동반 하락

그래서 이 스킬의 기본 자세는 **"검수자가 읽고 바로 발행 결재할 수 있는 수준"** 이다.

**이 단계를 건너뛰면 글은 반드시 얄팍해진다.** 자료 읽기를 충분히 한 뒤에만 써라.

### 0-1. 기존 가이드 톤 샘플 (최소 2편)

```bash

```

이미 발행된 글 중 **가장 최근 2편**을 읽어 톤·구조·어휘를 체화한다. 새 글이 기존 글과 **동일한 감정 온도**를 가져야 한다.

### 0-2. 카드 해석 글이면 - 원본 해석 전부 읽기

```
Read /home/tjd618/lovtaro/src/data/cardDictionary.js  (해당 카드 블록 전체)
```

- `upright.core`, `upright.love`, `upright.advice`
- `reversed.core`, `reversed.love`, `reversed.advice`

이미 쓰여 있는 문장을 **외우다시피 읽는다**. 가이드 글은 이 내용을 *반복*하는 게 아니라 *심화*해야 한다. 중복 문장이 나오면 독자는 "카드 페이지와 똑같은 걸 또 봤다"고 느낀다.

**역할 분담 (2026-04-19 moon 가이드 리라이트 회고)**:

카드 상세 페이지와 가이드 글은 역할이 달라야 한다. 겹치면 SEO·UX 둘 다 손해.

| 영역 | 카드 상세 (`cardDictionary.js`) | 가이드 글 |
|------|------|------|
| 시각·상징 묘사 (두 개의 탑, 늑대, 반사된 빛 등 그림 요소) | **여기서만** | 언급 금지 또는 최소화 |
| 상황별 짧은 한 문장 해석 | `upright.love` 안에서 | 그대로 재서술 금지 |
| 심리·루틴·행동 패턴 (새벽 감정, SNS 확인, 선택적 기억 등) | 간략 | **여기서 깊게** |
| 자기 점검 질문, 구체 실천법 | 없음 | **여기서만** |
| 카드 조합 해석 | 없음 | **여기서만** |

**카드 해석 가이드를 쓸 때의 원칙**: 카드 본문이 "이 카드는 무엇인가"라면, 가이드는 "이 카드가 나왔을 때 나는 무엇을 해야 하나"를 다룬다. 시각 묘사로 서두를 여는 구조는 피한다 - 독자가 카드 페이지와 같은 문장을 먼저 마주치게 됨.

## 2단계. 제목·메타 카피

### 제목 공식

구글 검색에서 **실제로 클릭을 받는** 제목을 쓴다. 너무 시적이면 CTR이 낮고, 너무 평범하면 AI스럽다.

**형식 A (카드 해석 글)** - `{카드 이름} {연애/타로} {핵심 키워드} - {부제}`
- 좋음: "달 카드 연애 해석 - 불안한 감정의 신호를 읽는 법"
- 좋음: "연인 카드 의미 - 진짜 선택이 다가왔다는 신호"
- 나쁨: "달 카드의 모든 것 완벽 정리" (AI 패턴)
- 나쁨: "달, 밤에 빛나는 이야기" (검색 키워드 부족)

**형식 B (상황 글)** - `{상황/질문} 타로 - {핵심 관점}`
- 좋음: "썸남 마음 보는 타로 - 연락 패턴에서 읽는 진심"
- 좋음: "재회 타로에서 자주 나오는 카드 조합 7가지"
- 나쁨: "썸남 마음이 궁금할 때" (타로 키워드 없음)

**형식 C (방법 글)** - `{방법} - {난이도/결과}`
- 좋음: "타로 3장 스프레드 해석법 - 과거·현재·미래 연결하는 법"
- 좋음: "역방향 타로 읽는 법 - 정반대가 아닌 이유"

### 길이 기준

- **title**: 25-40자 한국어 (Google 모바일 절삭 기준 안쪽)
- **description**: 70-120자. 글이 다루는 범위 + 독자가 얻는 것 + 자연스러운 호기심 유발
- **설명 금지어**: "완벽 가이드", "총정리", "한눈에", "모든 것"

> 달 카드를 뽑았을 때 자주 보이는 풍경이 있어요. 연락은 끊기지 않았는데 대화가 얕아진다거나, 상대의 한 마디가 유독 오래 마음에 남는 시기. 달빛이 그렇듯, 보이는 것의 윤곽이 흐려지는 때예요.

### 4-4. 단정 금지

카드 해석은 가능성의 언어다. 결정론적 표현 일체 금지.

| 금지 | 대체 |
|------|------|
| 반드시 | 많은 경우, 대체로 |
| 100% | 뚜렷하게, 높은 확률로 |
| 절대 | ~하지 않는 편이 자연스러워요 |
| 무조건 | 대부분 |
| 보장됩니다 | 예상돼요 |
| 틀림없이 | 분명하게 느껴지기로는 |

### 4-5. AI 패턴 전면 금지

도입부 첫 문장에 다음이 나오면 **즉시 리라이트**:
- "안녕하세요", "반갑습니다"
- "오늘은 ~에 대해 알아보겠습니다"
- "이번 글에서는 ~을 정리해드립니다"
- "~에 대해 살펴보겠습니다"

글 어디에도 나오면 안 되는 것:
- "정리합니다", "완벽 가이드", "한눈에 정리", "총정리"
- "결론적으로" (굳이 말하지 않아도 됨)
- "~해드리겠습니다" (과한 겸양)
- "이시군요!", "그렇군요!" (AI 과잉 공감)

### 4-6. em dash(—) 전면 금지

전 프로젝트 공통 규칙. 필요 시 하이픈(-), 불필요하면 삭제. 쉼표·마침표로 대체 가능하면 그쪽 우선.

### 4-7. 자극·불안 마케팅 금지

- "이 사람과 헤어집니다", "큰일 나기 전에", "끔찍한 일이", "저주받은 관계"
- 타로를 통해 **공포를 팔지 않는다**. 러브타로 브랜드 톤 훼손.
- 역방향 카드 해석도 "잠시 돌아볼 때"로 열어둘 것.

### 4-8. 자가 부정 금지

"타로는 미신이지만", "재미로 보세요" 같은 표현은 본문에 넣지 않는다. 면책은 `/disclaimer` 페이지가 담당. 본문에서 자가 부정하면 브랜드 일관성 훼손.

### 4-9. 반복·중복 없는 문장

글 내에서 같은 의미의 문장이 두 번 나오면 하나 삭제. 카드 페이지(`cardDictionary.js`)의 `love` 문장과 **60% 이상 겹치는 문장** 금지 - 독자가 같은 글을 두 번 읽은 느낌을 가진다.

### 4-10. 구체성 점수

각 섹션을 쓴 뒤 자문: "이 문단에 **이 카드에만** 해당하는 구체적 디테일이 있나?"
- 있다면 OK
- "이 카드 저 카드 다 쓸 수 있는 문장뿐"이면 실패. 다시 쓴다.

FAQ는 JSON-LD `FAQPage`로 들어가고 Google Rich Result에 노출된다. 퀄리티가 가장 눈에 띄는 파트.

### 질문 만드는 법

검색 쿼리를 상상한다. 독자가 실제로 구글에 치는 문장:

좋은 질문 예시:

나쁜 질문:
- "달 카드의 모든 것은?" (추상적)
- "타로란?" (글 주제와 동떨어짐)

### 답변 작성

- 각 답변 **2-4문장, 150-250자**

- 너무 짧으면 JSON-LD 노출 감점

좋은 답변 예시:
> 꼭 이별을 의미하지는 않아요. 역방향 연인 카드는 두 사람 사이의 **선택의 혼란**을 나타내는 경우가 더 많아요. 가치관이 다른 지점에서 부딪치고 있거나, 결정을 미뤄온 문제가 수면 위로 올라오는 시기예요. 지금 필요한 건 이별이 아니라 솔직한 대화일 수 있어요.
